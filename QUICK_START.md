# Riverchain 智能投资决策系统 - 快速开始指南

## 🚀 5 分钟快速上手

### 第 1 步: 访问应用
在浏览器中打开以下地址之一：
- **本地**: http://localhost:3000
- **公共访问**: https://3000-ituehabhum05ormfvf7pa-c81df28e.sandbox.novita.ai

### 第 2 步: 配置 API Key (必须)
1. 点击页面右上角的 **"设置"** 按钮
2. 在弹出的对话框中输入你的 Perplexity API Key
3. 选择模型（建议使用 **sonar-pro**）
4. 点击 **"保存设置"**

> 💡 **获取 API Key**: https://www.perplexity.ai/settings/api

### 第 3 步: 填写借款方信息
在"借款方信息"标签页中填写：
- ✅ 公司名称（必填）
- ✅ 运营年限（必须 ≥ 5年）
- ✅ 年收入（必须 ≥ HK$ 10M）
- 毛利率、净利润率
- 流动比率、速动比率
- 资产负债率
- 业务类型、法律地位
- 是否存在诉讼记录

### 第 4 步: 填写交易结构
切换到"交易结构"标签页：
- ✅ 申请融资金额
- ✅ 融资类型（保理/发票融资/贷款）
- ✅ 发票金额
- ✅ 对手方名称
- 对手方类型（港府/MTR/地产商/其他）
- 账期

系统会自动计算 LTV 比率并检查合规性。

### 第 5 步: 添加文档上下文（可选）
在"文档上下文"标签页粘贴：
- 财务报表内容
- 公司简介
- 其他相关文档

> 💡 提供更多背景信息可提高评估准确性

### 第 6 步: 配置自定义智能体（可选）
如需额外的专家视角：
1. 在"自定义智能体配置"区域点击 **"新增智能体"**
2. 输入名称（如"宏观经济专家"）
3. 编写角色设定（System Prompt）
4. 添加补充知识库（可选）
5. 点击 **"添加"**

示例智能体配置：

**名称**: 宏观经济专家

**角色设定**:
```
你是一位宏观经济分析师，专注于评估香港建筑行业的市场周期和宏观风险。
请基于当前经济环境（房地产市场、利率走势、政府基建投资计划等）
评估该贷款申请的宏观风险，并提供你的独立意见。
```

### 第 7 步: 开始评估
1. 点击"评估控制"区域的 **"开始评估"** 按钮
2. 系统将同时运行核心分析师和所有启用的自定义智能体
3. 等待评估完成（通常 30-60 秒）

### 第 8 步: 查看结果
评估完成后，您将看到：

#### 📊 决策结果
- **批准 (APPROVE)**: 绿色，风险可控
- **拒绝 (REJECT)**: 红色，风险过高
- **有条件批准 (CONDITIONAL)**: 黄色，需要额外措施

#### 📈 评分卡
- 综合评分：0-100 分
- 风险等级：低/中/高
- Stage 1 得分（商业实体）
- Stage 2 得分（交易质量）

#### ⚠️ 红旗警示
如果触发任何红旗条件，将醒目显示：
- 运营年限不足
- 年收入不达标
- 存在诉讼记录
- LTV 超限
- 集中度超限

#### 📋 详细分析报告
完整的 Markdown 格式报告，包含：
- 红旗检查结果
- Stage 1 评估详情
- Stage 2 评估详情
- 综合评分说明
- 风险等级判定
- 具体建议措施

## 📝 示例场景

### 场景 1: 正常评估（应该批准）
```
借款方信息:
- 公司名称: 香港建筑工程有限公司
- 运营年限: 10 年
- 年收入: HK$ 50M
- 毛利率: 18%
- 净利润率: 6%
- 流动比率: 2.0
- 速动比率: 1.5
- 资产负债率: 45%
- 诉讼记录: 无

交易结构:
- 融资金额: HK$ 8M
- 融资类型: 保理业务
- 发票金额: HK$ 10M
- 对手方: 港铁公司 MTR
- 对手方类型: MTR
- 账期: 60 天

预期结果: ✅ 批准，低风险，评分 85+
```

### 场景 2: 触发红旗（应该拒绝）
```
借款方信息:
- 公司名称: 新成立建筑公司
- 运营年限: 3 年 ⚠️
- 年收入: HK$ 8M ⚠️
- 诉讼记录: 是 ⚠️

预期结果: ❌ 拒绝，高风险
红旗: 运营年限不足、年收入不达标、存在诉讼记录
```

### 场景 3: LTV 超限（应该拒绝或有条件批准）
```
交易结构:
- 融资金额: HK$ 9M
- 融资类型: 发票融资
- 发票金额: HK$ 10M
- LTV: 90% ⚠️（超过 70% 上限）

预期结果: ❌ 拒绝 或 ⚠️ 有条件批准
```

## 🎯 评估标准速查表

### 红旗警示（自动拒绝）
- ❌ 运营年限 < 5年
- ❌ 年收入 < HK$ 10M
- ❌ 存在诉讼记录
- ❌ LTV 超限

### LTV 限制
| 融资类型 | 上限 |
|---------|------|
| 保理业务 | 90% |
| 发票融资 | 70% |
| 贷款 | 根据情况 |

### 风险等级
| 分数 | 风险等级 | 决策 |
|-----|---------|-----|
| 80-100 | 低风险 | 批准 |
| 60-79 | 中等风险 | 有条件批准 |
| 0-59 | 高风险 | 拒绝 |

### 对手方评级
| 类型 | 评级 | 示例 |
|-----|------|-----|
| 港府机构 | A 级（优质） | 房屋署、路政署 |
| MTR | A 级（优质） | 港铁公司 |
| 大型地产商 | A 级（优质） | 新鸿基、信和、恒基 |
| 其他 | 需评估 | 中小企业 |

## 💡 使用技巧

1. **首次使用**: 建议先用示例场景 1 测试系统是否正常工作
2. **API Key**: 保存后刷新页面不会丢失，只需配置一次
3. **文档上下文**: 提供越详细的背景信息，评估越准确
4. **自定义智能体**: 可以添加多个不同视角的专家
5. **并行评估**: 所有智能体会同时工作，节省时间
6. **报告保存**: 可以复制 Markdown 报告到其他文档

## ⚠️ 常见问题

### Q1: 点击"开始评估"没有反应？
**A**: 检查是否已配置 Perplexity API Key

### Q2: 评估失败，显示错误？
**A**: 
- 检查 API Key 是否有效
- 检查网络连接
- 确认 Perplexity API 配额是否充足

### Q3: 评估时间过长？
**A**: 
- sonar-pro 通常 30-60 秒
- sonar-reasoning 可能需要 1-2 分钟
- 多个自定义智能体会延长总时间

### Q4: 如何解读"有条件批准"？
**A**: 表示贷款可以批准，但需要：
- 额外的担保措施
- 更严格的监控
- 降低融资金额
- 具体建议见"建议措施"部分
